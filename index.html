regra da cascata no css

o css segue a logica da cascata: qnd duas regra conflitam, a q tem maior especificidade ou é lida por ultimo vence

especificidade

a especificidade define qual regra tem mais prioridade

  • seletor por tag → menor peso
  • seletor por classe → peso medio
  • seletor por id → maior peso

combinação de seletores

.primeiro h1 → pega todos os h1 dentro de .primeiro

.primeira > h1 → pega so os filho direto h1

.primeira h1 + h1 → pega o h1 logo depois de outro h1

pseudo-classes

:first-child → pega o primeiro filho

:last-child → pega o ultimo filho

:nth-child(n) → pega o filho de numero n

:nth-of-type(n) → pega o filho de um tipo especifico

display inline x block

inline

n quebra a linha, so aceita margem horizontal. exemplo: <span>, <a>, <strong>

exemplo: sou um span to na mesma linha

block

quebra a linha, aceita width e height. exemplo: <div>, <p>, <h1>

sou um bloco
ocupo a linha toda

box model: margin, padding e border

margin: espaço de fora

padding: espaço de dentro

border: a borda msm

shorthand

  • 1 valor → todos os lados
  • 2 valores → top/bottom e left/right
  • 3 valores → top, left/right, bottom
  • 4 valores → top, right, bottom, left

box-sizing

box-sizing: border-box; → largura ja inclui conteudo + padding + borda
